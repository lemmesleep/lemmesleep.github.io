Basic operations

1. Creating, deleting and altering a table

They are all DDL commands, because these commands influence the structure of the database component.

general keywords

  • CREATE TABLE [table_name]; : DDL command which create the structure of a table
    • while creating a table, some constraint can also be applied to the end the command datas such as
      • AUTO_INCREMENT: incrementing the value each time a new row is created
      • NOT NULL : cannot be set to null
      • DEFAULT [default_setting]: value to assign if it is null
  • DESCRIBE: displaying the information of the table
  • ALTER TABLE [name of the table] ADD [name of column] [type]: add a column
  • ALTER TABLE [table_name] DROP COLUMN [column name]: remove a column
  • DROP TABLE [table_name]; : DDL command which deletes the entire table structure (including the columns of the table).

2. Inserting, updating, and deleting and selecting data from a table

They are DML commands because they manipulate the datas inside of a table; selecting data(fetch) can also refer to the DQL, since it is a command for Queries.

Inserting data into the table

general command:

  • INSERT INTO [table name] VALUES([values according to the columns])

sometimes we only want to define few of the values:

  • INSERT INTO [table name] ([name of the columns to be defined]) VALUES([values])

Deleting data from the table

DELETE FROM[table name]: delete the data from a table

  • it is different from DROP because it only affect the data in the table, deleting has no impact on the structure.

Updating data in the table

general command:

UPDATE [table]

SET [to_be_changed_column] = ‘new_value’

(optional)

WHERE [target_column] = ‘value’

  • keyword such as OR and AND can be used on top of the WHERE statement.
  • when the WHERE keyword is not used, the update applies to all table
  • see the WHERE keyword as an if statement.

Getting information from the database(Queries)

SELECT [attribute(s)] or *

FROM [table]

WHERE [target_column] = ‘value’;([target_column] IN (these values))

ORDER BY [attribute] DESC or ASCE;

LIMIT [number to limit];
